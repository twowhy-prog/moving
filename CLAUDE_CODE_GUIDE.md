## 2. Claude Code 시작 프롬프트 (복붙용)

아래 내용을 Claude Code 첫 메시지로 그대로 붙여넣으세요.

```
다음 작업을 수행해줘.

[배경]
부동산 매도·매수 및 재무설계 내용을 담은 홈페이지가 있어.
financial_data.json 파일에 최신 수치와 전략이 담겨 있어.
이 데이터를 기반으로 홈페이지의 관련 섹션들을 업데이트해줘.

[작업 순서]
1. financial_data.json 읽기
2. 기존 홈페이지 파일 구조 파악 (어떤 파일에 어떤 섹션이 있는지)
3. 아래 업데이트 항목별로 수정

[업데이트 항목]
- 매도 관련: sale_price, transfer_tax, proceeds_after_all
- 신규매수 관련: options[0](A물건), options[1](B물건) 비교표
- 대출 규제: loan_regulation 내용
- 코인 전략: crypto_strategy 테이블
- 교육비: education_cost 단계별 표
- 타임라인: timeline 항목들
- 지역분석: location_analysis.norangjin, isu
- 체크리스트: checklist 항목들

[주의사항]
- 숫자는 한국식 단위(억, 만원)로 변환해서 표시
- 금액이 0 이상인 항목만 표시
- 기존 디자인·레이아웃은 유지하고 텍스트/수치만 교체
- 변경된 파일 목록과 변경 내용 요약을 마지막에 알려줘
```

## 4. 섹션별 Claude Code 명령어

### 매도 섹션만 업데이트
```
financial_data.json의 current_home 데이터로
홈페이지의 "매도 계획" 섹션만 업데이트해줘.
transfer_tax.combined와 proceeds_after_all 수치를 반영하고
잔금일 관련 주의사항(2026-08-02 이후 설정 시 절세 효과)도 추가해줘.
```

### 두 물건 비교표 업데이트
```
financial_data.json의 new_home.options 배열을 읽어서
A물건(34평)과 B물건(45평) 비교표를 업데이트해줘.
B물건의 영유비 포함 주담대가 4억 한도를 초과한다는 경고도 표시해줘.
```

### 코인 전략 섹션 업데이트
```
financial_data.json의 crypto_strategy 배열과
mortgage_payoff_scenarios 배열을 읽어서
코인 활용 전략 및 주담대 조기상환 시나리오 표를 업데이트해줘.
2027년 과세 시행 관련 주의문구도 추가해줘.
```

### 타임라인 업데이트
```
financial_data.json의 timeline 배열을 읽어서
인생 재무 타임라인 섹션을 업데이트해줘.
각 항목의 year, event, action을 반영하고
2031년(노량진 완성)과 2038년(고등학교·최대지출)을 강조 표시해줘.
```

### 지역 분석 업데이트
```
financial_data.json의 location_analysis를 읽어서
노량진 뉴타운과 이수역 섹션을 업데이트해줘.
price_appreciation_scenarios 표와
relocation_strategy 표를 함께 업데이트해줘.
```

## 5. 단위 변환 규칙 (Claude Code에게 전달)

```
숫자 표시 규칙:
- 1,000,000,000 이상 → X억 Y,000만원
- 100,000,000 이상 → X억
- 10,000,000 이상 → X,XXX만원
- 1,000,000 이상 → X,XXX,XXX원
- 음수(지출)는 -X억 또는 -X,XXX만원으로 표시
- 퍼센트는 소수점 1자리까지

예시:
1,478,100,000 → 14억 7,810만원
42,403,842 → 약 4,240만원
1,663,000 → 166만 3천원
0.007 → 0.7%
```

## 8. 주의사항

- `financial_data.json`은 **읽기 전용 소스**로 취급 (수정 금지)
- 홈페이지 수정 전 반드시 **git commit 또는 백업** 권장
- 금융 수치는 **반올림 없이** 원본 그대로 사용
- 법적 면책 문구는 반드시 유지: *"본 내용은 참고용이며 최종 결정 전 전문가 확인 필수"*
